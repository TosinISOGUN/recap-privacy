What Recap does

Recap reads the Jira issues assigned to you that were completed in the last 30 days, groups them by epic (or project), and writes a short, plain-language status report you can paste straight into a status update, a 1:1 doc, or a performance review. No invented metrics, no filler — just an honest summary of what actually shipped, in words.

It runs entirely on Atlassian Forge. The report is written using Atlassian's own AI, so your data never leaves the Atlassian platform, and Recap stores nothing.

How to use it

  1. In Jira, open Apps in the top navigation and select Recap.
  2. Click Generate my report.
  3. Your report appears in a text box. Click into it, select all, and copy — edit anything first if you like.

Frequently asked questions

Which work does Recap include?

Issues assigned to you that moved to a “Done” status in the last 30 days. The window is fixed at 30 days.

It says “Nothing completed in the last 30 days.”

That means no issues assigned to you reached a Done status in that window. Recap won’t invent a report — complete some work and generate again.

It showed a “Basic report” notice.

If the AI writer is momentarily unavailable, Recap falls back to a plain summary of your completed work so you’re never left empty-handed. Try again in a moment for the written version.

Why does it group work under a project name instead of an epic?

Issues that aren’t linked to an epic (no parent) are grouped under their project instead, so nothing is dropped.

What data does Recap access or store?

It reads only your completed issues’ titles, epic/project, and completion date, using a single read-only permission — and it stores nothing. See the privacy policy for detail.

How do I copy the report?

Click inside the report box, press Ctrl/Cmd + A to select all, then Ctrl/Cmd + C to copy.
